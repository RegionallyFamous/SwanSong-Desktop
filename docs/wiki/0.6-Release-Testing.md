# 0.6 Release Testing

SwanSong 0.6.1 makes the workshop around the player much more capable. Studio
gets the full SDK 0.5 toolset, Story Forge becomes a nine-part writing room, and
Translation Lab can prove that private source evidence belongs to the exact
captured native frame before provenance begins.

Test only games, manuscripts, homebrew, SDK projects, cartridges, and save data
you own or are authorized to use. Never attach ROMs, saves, private screenshots
or audio, manuscripts, cartridge-source evidence, bearer tokens, or project
evidence to a public report.

## Studio

- create and reopen Game and Utility App projects;
- move through New, Assets, Build, Test, Play, Profile, Evidence, and Release;
- exercise traced builds, scenario compilation, all-contract play, semantic
  outcomes, budget history, project migration, and hardware capacity;
- review asset and audio changes before applying them;
- confirm incomplete or stale PNG/WAV evidence blocks Release; and
- restart local MCP and confirm it exposes the complete guarded tool set
  without asking for a login-keychain password.

Python 3.11+ and Wonderful remain separate prerequisites. Doctor should explain
what is missing without pretending the dependency is bundled.

## Story Forge

- prepare the eight proposal-only Story Room packets and confirm no role edits
  the manuscript, canon, approval, or release state;
- build the story map, edit a chapter, and verify Save and Discard are explicit;
- refresh live context for a stable scene ID;
- create and compare an immutable revision snapshot, then append a decision;
- export an unprimed reader packet, import a consented current response, and
  reject stale, incomplete, anonymous, or duplicate responses;
- create and check research, genre, Art Room, and Music Room artifacts;
- compile the source-mapped `.wscvn` scaffold, confirm it remains explicitly
  non-production-ready, and detect drift after either source changes; and
- continue into Studio without losing the open novel.

Private manuscripts, packets, maps, revisions, reader responses, research,
art, music, adaptations, approvals, and editions stay in selected local
folders. Automation reports coarse status and navigation only.

## Translation Lab

- retain deterministic routes, Original/Patched captures, observed-play
  recovery, owner/source probes, and source-free evidence browsing;
- run the guarded capture-plan contract and confirm only the exact private
  capture outputs are accepted;
- prove the expected native frame number and framed fingerprint are validated
  before owner or source queries and revalidated afterward;
- verify the private native query receipt has the order **frame validated →
  owner query started → source query started → frame revalidated**;
- change the expected frame or fingerprint and confirm no report, private
  details, run directory, output, or closure is created; and
- confirm MCP returns only source-free hashes, counts, geometry, component
  summaries, and completeness while exact addresses, lineages, ranges, writers,
  and outside consumers remain private.

## Everyday Mac experience

- add and play authorized WonderSwan, WonderSwan Color, SwanCrystal, and Pocket
  Challenge V2 software without a BIOS;
- verify keyboard and controller input, portrait play, Time Ribbon, visual save
  states, Pocket save exchange, and native 224×157 fitting;
- inspect Finder, Dock, direct-launch, and menu-bar swan artwork in light and
  dark appearances; and
- check Library, Homebrew, Translation Lab, Analogue Pocket, Studio, Story
  Forge, Cartridge Lab, Settings, Updates, and Support at their real sizes.

## Release and update

- confirm **SwanSong 0.6.1 (12)** in About SwanSong;
- verify GitHub identifies 0.6.1 as the Latest stable release;
- prove the normal update channel offers 0.6.1 to 0.6.0;
- confirm the signed archive, corresponding-source archive, manifest, and
  checksums agree;
- verify Developer ID, notarization, stapling, Gatekeeper, arm64, and x86_64;
- retain the prior app after an unreachable feed, bad signature, interrupted
  download, cancellation, or failed installation; and
- confirm updater traffic contains no system profile or per-user identifier.

The Homebrew Catalog stays offline until an explicit Browse, Refresh, or Add
action. The Analogue Pocket workflow stays locked until the separate SwanSong
Core project publishes an authorized stable release.

## Reporting checklist

1. Record Mac model, macOS version, architecture, controller or cartridge
   hardware when relevant, and exact SwanSong version/build.
2. Report the visible workspace, action, and sanitized result without private
   project material.
3. For provenance, report only frame, rectangle dimensions, selected
   components, completeness, and source-free counts.
4. Distinguish **Picture Appeared** from an end-to-end **Works** verdict.
