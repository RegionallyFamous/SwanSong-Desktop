# 0.4 Beta Testing

Test only game and homebrew images you own or are authorized to use. Never
attach ROMs, saves, private screenshots, audio captures, or project evidence
to a public report.

## Primary test areas

- select a local SwanSong SDK 0.2.0-or-newer checkout and verify Doctor reports
  the SDK, schema, Wonderful, SwanSong, Python, paths, and project configuration;
- create fresh Arcade Action, Menu Puzzle, and Grid Tactics projects, then move
  each through Assets, Build, Test, Play, Profile, Evidence, and Release;
- run Dev, trigger source and asset rebuilds, relaunch through SwanSong, and
  confirm Cancel leaves no watcher or child process behind;
- convert an actual-input export with Scenario Recorder, edit the frame plan,
  and prove repeated play produces deterministic state and evidence;
- exercise deterministic fuzzing, Sprite/VRAM profiling, Asset Optimizer, and
  every Save/RTC Laboratory fault mode;
- create intentional PNG and WAV changes, inspect them in Evidence Diff, and
  confirm an unreviewed, incomplete, or stale evidence set blocks Release;
- retain normal Open IPL, imports, play, controller, Time Ribbon, Translation
  Lab, local MCP, and updater coverage on Apple silicon and Intel.

## Guardrails

- Studio invokes exact SDK commands and rejects unknown structured-result
  schemas. It does not reinterpret `swan.toml`, compile assets itself, or own a
  second release policy.
- SwanSong is the only gameplay-validation backend. No alternate emulator may
  appear in commands, reports, documentation, or acceptance evidence.
- SDK and project folders must be explicitly selected. Private source, ROM,
  save, state, persistence, diagnostics, capture, and audio bytes stay local.
- Only one SDK command may run at once. Cancellation must terminate its entire
  process group before another command begins.
- Release requires inspected, current, hash-bound observations for every Play
  Contract check. Execution success or changing hashes alone are insufficient.

## Expected boundaries

This developer preview does not bundle Python, Wonderful, or SwanSong SDK.
Choose an SDK 0.2.0-or-newer checkout whose dependencies are already installed.

The Homebrew Catalog remains **Coming Soon** and network-silent. The Analogue
Pocket tool remains locked until a verified stable Core release is published.
Raw HID mappings are not guessed; physical input and signed update installation
still require hands-on testing.

## Reporting checklist

1. Confirm **SwanSong 0.4.0 (6)** in About SwanSong.
2. Record Mac model, macOS version, architecture, SDK version, Python version,
   and Wonderful revision.
3. Include the Studio workspace, exact visible command, project recipe, and
   whether a new generated project reproduces the issue.
4. For Play, Evidence, or Release failures, include the scenario/check name and
   sanitized status without attaching private media, paths, or identifiers.
5. For Dev or cancellation failures, say whether a child process remained.
6. Distinguish **Reached Video** from an end-to-end **Works** verdict.
