# 0.4 Beta Testing

Test only game and homebrew images and SDK projects you own or are authorized
to use. Never attach ROMs, saves, original firmware, private screenshots, audio
captures, Translation Lab evidence, or proprietary project assets to a public
report.

## Primary test areas

- choose a local SwanSong SDK checkout, create Arcade Action, Menu Puzzle, and
  Grid Tactics projects, then exercise all eight SwanSong Studio workspaces;
- confirm SwanSong Studio retains diagnostics and shows generated controls,
  assets, resource budgets, native PNG/WAV evidence, exact input plans, and
  resolved SDK, Wonderful, schema, and engine identities;
- probe a translated rectangle upstream and confirm exact or conservative
  cartridge ranges, per-display chains, and outside consumers stay private;
- restart Codex and confirm the installed plugin exposes the two playtest tools
  and all fourteen Translation Lab and observed-play tools;
- inspect SwanSong's status item in light and dark menu bars and test Show
  SwanSong and Quit SwanSong;
- update from 0.3 with **Include beta versions** enabled and confirm the library,
  saves, states, preferences, and private projects are preserved; and
- retain normal Open IPL, import, player, controller, Time Ribbon, display,
  portrait, accessibility, and relaunch coverage on Apple silicon and Intel.

## Automation and privacy guardrails

- SwanSong Studio must invoke the selected SDK's exact `swan` commands and must not
  implement a second manifest parser, asset converter, budget policy, Wonderful
  build, or acceptance emulator inside Desktop.
- Doctor, Optimizer, Fuzzer, Save/RTC Lab, Scenario Recorder, Dev, Profile,
  Evidence Diff, and Release must reject unknown structured result schemas.
- The SwanSong Studio developer preview must disclose that the tagged SDK, Python
  runtime, and deterministic play executor are not bundled in this package.
- `probe-rectangle-source` rejects invalid or oversized native rectangles.
  Exact and candidate cartridge ranges, source addresses, coordinates,
  per-display chains, and outside consumers remain private; MCP receives only
  hashes, counts, and completeness.
- Unknown, carry-dependent, ambiguous, or overflowing upstream dataflow is
  marked incomplete instead of exact.
- Persisted capture, display-owner probes, route record/verify, and observed
  play retain their 0.3 authorization, path-containment, bounded-write, empty
  persistence, fixed RTC, and clean-boot replay contracts.
- Live MCP remains off by default, uses a user-only bearer token, exposes no
  game title or path, and revokes its token when disabled.
- No MCP or Translation Lab error may return ROM, save, state, persistence,
  RAM, cartridge bytes, or unapproved framebuffer data.

## Expected unavailable features

- SwanSong Studio is not yet a self-contained public SDK distribution. Select a
  local SDK checkout and install the pinned Wonderful toolchain for this beta.
- The Homebrew page says **Coming Soon** and performs no catalog or game
  download. Use **Add From Mac** for authorized local homebrew.
- The Analogue Pocket tool reports that no verified Core release is published
  and performs no package download or card write.
- MCP is local developer automation, not a public network or remote-control
  service. Mutating tools still require client approval.

## Feature freeze

The 0.4 beta feature set is frozen after SwanSong Studio, ABI 7 upstream source
provenance, full plugin exposure, and the SwanSong status item. Remaining 0.4
work is limited to release-blocking fixes, tests, documentation, signing,
notarization, packaging, update verification, and publication.

## Reporting checklist

1. Confirm **SwanSong 0.4.0 (6)** in About SwanSong.
2. Record Mac model, macOS version, architecture, and controller when relevant.
3. For SwanSong Studio, name the action, SDK version, manifest schema, and sanitized
   visible error without attaching source, assets, ROMs, paths, or evidence.
4. For provenance, report the frame, rectangle dimensions, completeness, and
   source-free counts only.
5. For MCP, state whether SwanSong and Codex were restarted, whether local
   control was enabled, the tool name, and the sanitized visible error.
6. For updater issues, include installed and offered version/build, channel,
   opt-in settings, and exact visible error. Do not attach a rejected archive.
