## SwanSong 0.6.1 — Open and Go

SwanSong 0.6.1 is a small update with a big goal: get you into the app without
making you wait, explain yourself to a password box, or convince Studio that
the modern Python already installed on your Mac really exists.

Expected app identity: **SwanSong 0.6.1 (12)**.

## Before you install

SwanSong requires macOS 14 or later. The official universal app supports Apple
silicon and Intel Macs. It includes no games or original system firmware; use
only game, manuscript, project, and cartridge material you own or may handle.

This is a stable release. Existing users can install it through **Check for
Updates…** after the signed update feed is published. For a manual install,
verify `SwanSong-0.6.1-macOS-universal.zip` against `SHA256SUMS.txt` and the
matching release manifest from the same GitHub release.

## SwanSong gets out of bed faster

The main window now appears before SwanSong prepares the library, controllers,
local tools, and updater. Studio and Story Forge wait until you actually open
them. Your collection and workshop still finish waking up in the background,
but the Mac no longer has to hold the door shut while that happens.

## No surprise catalog password box

An older Homebrew Catalog trust record could ask for the Mac login password
after an app update. SwanSong now reads that tiny anti-rollback record without
allowing macOS to interrupt launch. New records are tied to SwanSong's signed
app identity so future signed updates keep the same safe access.

This changes no games, saves, projects, accounts, or update settings. SwanSong
still has no accounts, analytics, telemetry, or background catalog browsing.

## Studio finds the Python you already installed

Opening a Mac app from Finder gives it a much smaller command search path than
opening a shell. Studio could therefore find Apple's older Python and miss a
newer Homebrew installation, then report the bundled SDK as damaged.

Studio now checks the standard Homebrew, python.org, MacPorts, and system
locations directly and selects Python 3.11 or newer. If no compatible runtime
exists, it gives a clear installation message instead of blaming the SDK.

## A quicker road to the next release

Documentation-only and signed-update-feed changes now use a seconds-long CI
lane while source changes still receive the full Mac test matrix. The release
tool also checks notarization credentials before starting its long sealed
build, so a missing or locked profile fails immediately.

## What stays the same

SwanSong 0.6.1 keeps the complete SDK 0.5 Studio, Story Forge, Translation Lab,
Homebrew Catalog, Cartridge Lab, visual library, Time Ribbon, save states,
controller support, and crisp native player introduced across the 0.6 line.

- Studio still needs Python 3.11+ and the pinned Wonderful packages for builds.
- Story Forge still needs a compatible local schema-v3 framework checkout.
- The separate Analogue Pocket Core remains unavailable until its own project
  publishes an authorized stable release.
- No games, original firmware, private evidence, manuscripts, or cartridge
  contents are included in the app.

When reporting an issue, include SwanSong 0.6.1 build 12, macOS version, Mac
architecture, and the visible action and message. Never attach ROMs, saves,
private screenshots or audio, manuscripts, project paths, bearer tokens, or
Translation Lab evidence.
