## SwanSong 0.4.0 beta — Build Where You Play

SwanSong 0.4 opens the door from playing WonderSwan games to making and
investigating them. SwanSong Studio brings the SDK workflow into the Mac app,
while Translation Lab gains a deeper, privacy-preserving path from visible
pixels toward the cartridge data that produced them.

### Make games in SwanSong

- **Start from a real recipe.** Create Arcade Action, Menu Puzzle, or Grid
  Tactics projects without hand-building a directory.
- **Keep the whole loop together.** New, Assets, Build, Test, Play, Profile,
  Evidence, and Release each have a focused SwanSong Studio workspace.
- **Find setup problems early.** Doctor reports the selected SDK, Wonderful,
  SwanSong, Python, paths, and project configuration before they become opaque
  build failures.
- **Use the complete tool suite.** Dev watches and rebuilds projects; Scenario
  Recorder converts actual input into editable plans; deterministic fuzzing,
  Sprite/VRAM profiling, Asset Optimizer, and Save/RTC Laboratory expose
  gameplay, hardware-budget, asset, persistence, and clock failures.
- **Require evidence, not a green command.** Evidence Diff compares inspected
  PNG output and meaningful WAV changes. Release requires current, hash-bound
  observations for every scenario check before the SDK packages ROMs, reports,
  checksums, and notes.

### Trace visible results upstream

- **Select the source that matters.** ABI 8 can seed map, raster, or palette
  components independently, so an unrelated component does not contaminate a
  source result.
- **Follow the cartridge lineage.** A bounded native rectangle can be traced
  through common copies and transforms to exact or honestly conservative
  cartridge ranges, selected display chains, and other display consumers.
- **Retain executed-read context.** Private evidence can include immediate
  caller and code location, operand segment and offset, mapper window and bank,
  and the resolved cartridge operand.
- **Keep private details private.** The guarded source-probe MCP tool returns
  only hashes, counts, selected-component summaries, and completeness. Exact
  offsets, emulated addresses, coordinates, chains, and executed-read details
  stay inside the Translation Lab project.

### A more complete SwanSong

- The installed MCP server exposes the complete guarded Translation Lab tool
  set, including persisted capture, both rectangle probes, record/verify, and
  the observed-play lifecycle.
- SwanSong has its own compact menu-bar status item with Show SwanSong and Quit
  SwanSong actions.
- Translation Lab uses its intended violet accent across the empty state,
  controls, selections, and text intake.

### One toolchain, clear ownership

Studio delegates its work to SwanSong SDK 0.2.0 or newer. The SDK owns
manifests, Wonderful builds, assets, deterministic scenarios, budgets, and
release policy; SwanSong remains the only gameplay and evidence backend. The
Desktop app does not create a competing build system or add another emulator.

Choose a local SDK checkout when Studio opens. This developer preview does not
yet bundle Python, Wonderful Toolchain, or the SDK runtime inside the signed
application.

### Private by design

Studio reads and writes only the SDK and project folders you select. Project
source, ROMs, saves, plans, diagnostics, frames, audio, profiles, and reports
stay on the Mac. Translation source-provenance details also remain in the
private project. A release verdict is not granted merely because execution
succeeded or a file hash changed.

### Before you install

SwanSong requires macOS 14 or later and supports Apple silicon and Intel Macs.
No games or original firmware are included. Add only game and homebrew images
and project material you own or are authorized to use.

This beta must be selected explicitly from GitHub Releases. Stable-only clients
must remain on their selected channel; beta-enabled clients can discover 0.4.0
after its signed, notarized prerelease and appcast entry are published. Verify
the universal app ZIP against `SHA256SUMS.txt` and the matching release manifest
before a manual install.

### Best beta targets

- Create, build, test, play, inspect, and package a fresh project from each SDK
  recipe.
- Start Dev, change source and assets, confirm the expected rebuilds, then
  cancel and verify no watcher or player process remains.
- Record a short input plan, edit it, replay it twice, and confirm the evidence
  and canonical state remain deterministic.
- Produce a real visual or audio regression and verify Evidence Diff flags it;
  confirm Release refuses uninspected or stale observations.
- Probe separate map, raster, and palette components and confirm both private
  lineage accuracy and source-free MCP responses.
- Restart the MCP client and confirm the complete guarded tool surface is
  present; inspect the swan status item in light and dark menu bars.

### Known limits

- Studio requires a user-selected SwanSong SDK 0.2.0-or-newer checkout plus its
  external Python and Wonderful dependencies.
- Commercial-title compatibility is still being validated. **Reached Video**
  is not an end-to-end verdict or original-hardware accuracy claim.
- Source provenance marks carry-dependent, ambiguous, unknown, or overflowing
  dataflow incomplete instead of claiming false exact dependencies.
- The Homebrew Catalog and SwanSong Core download paths remain unavailable
  until their independent signed publication gates pass.
