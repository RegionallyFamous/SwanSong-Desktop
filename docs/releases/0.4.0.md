## SwanSong 0.4.0 beta — A Workshop Inside SwanSong

SwanSong 0.4 adds SwanSong Studio, a native workspace for taking a WonderSwan
game from a new project to inspected release evidence. The player, deterministic
test backend, SDK tools, diagnostics, and release gates now meet in one place.

### What changed

- **Start from a real recipe.** Create Arcade Action, Menu Puzzle, or Grid
  Tactics projects through SwanSong SDK without hand-building a directory.
- **Keep the whole loop together.** New, Assets, Build, Test, Play, Profile,
  Evidence, and Release each have a focused Studio workspace.
- **Find setup problems early.** Doctor reports the selected SDK, Wonderful,
  SwanSong, Python, paths, and project configuration before they become opaque
  build failures.
- **Iterate while the game is moving.** Dev watches source and assets, rebuilds,
  and relaunches SwanSong through the SDK's controlled workflow.
- **Turn play into a test.** Scenario Recorder converts an exported button log
  into editable frame-plan JSON; the deterministic fuzzer searches for crashes,
  dead ends, invalid transitions, and reset divergence.
- **See where the handheld budget went.** Sprite/VRAM profiling exposes tile
  ownership, palettes, dirty regions, scanline pressure, and frame budgets.
  Asset Optimizer previews deduplication, palette reduction, flip reuse, and
  mono conversion before the source assets are changed.
- **Stress the hidden states.** Save/RTC Laboratory exercises corruption,
  interrupted writes, missing clocks, power loss, and time changes without
  risking a normal play save.
- **Require evidence, not a green command.** Evidence Diff compares inspected
  PNG output and meaningful WAV changes. Release requires current, hash-bound
  observations for every scenario check before the SDK packages the ROM,
  reports, checksums, and notes.

### One toolchain, clear ownership

Studio delegates its work to SwanSong SDK 0.2.0 or newer. The SDK continues to
own manifests, Wonderful builds, assets, deterministic scenarios, budgets, and
release policy; SwanSong remains the only gameplay and evidence backend. The
Desktop app does not create a competing build system or add another emulator.

Choose a local SDK checkout when Studio opens. This developer preview does not
yet bundle Python, Wonderful Toolchain, or the SDK runtime inside the signed
application.

### Private by design

Studio reads and writes only the SDK and project folders you select. Project
source, ROMs, saves, plans, diagnostics, frames, audio, profiles, and release
reports stay on the Mac. A release verdict records reviewed evidence identity;
it is not granted merely because execution succeeded or a file hash changed.

### Before you install

SwanSong requires macOS 14 or later and supports Apple silicon and Intel Macs.
No games or original firmware are included. Add only game and homebrew images
you own or are authorized to use.

This beta must be selected explicitly from GitHub Releases. Stable-only clients
must remain on their selected channel; beta-enabled clients can discover 0.4.0
after its signed, notarized prerelease and appcast entry are published. Verify
the universal app ZIP against `SHA256SUMS.txt` and the matching release manifest
before a manual install.

### Best beta targets

- Create, build, test, play, inspect, and package a fresh project from each of
  the three SDK recipes.
- Start Dev, change a source file and an asset, confirm the expected rebuilds,
  then cancel and verify no watcher or player process remains.
- Record a short real-input plan, edit it, replay it twice, and confirm the
  evidence and canonical state remain deterministic.
- Use the profiler and optimizer on a project near a declared resource limit,
  then confirm the build still enforces the manifest budget.
- Produce a real visual or audio regression and verify Evidence Diff flags it;
  confirm Release refuses an uninspected or stale observation.
- Exercise every Save/RTC fault mode without changing the project's ordinary
  save data.

### Known limits

- Studio requires a user-selected SwanSong SDK 0.2.0-or-newer checkout plus its
  external Python and Wonderful dependencies.
- Commercial-title compatibility is still being validated. **Reached Video**
  is not an end-to-end verdict or original-hardware accuracy claim.
- The Homebrew Catalog and SwanSong Core download paths remain unavailable
  until their independent signed publication gates pass.
