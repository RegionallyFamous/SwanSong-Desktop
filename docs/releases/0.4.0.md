## SwanSong 0.4.0 beta — Build Where You Play

SwanSong 0.4 opens the door from playing WonderSwan games to making and
investigating them. SwanSong Studio brings the SwanSong SDK workflow into the Mac
app, while Translation Lab gains a deeper, privacy-preserving way to follow a
visible result back toward the cartridge data that produced it.

### What changed

- **A full SwanSong Studio.** Eight focused New, Assets, Build, Test, Play,
  Profile, Evidence, and Release workspaces cover the complete SDK workflow
  without leaving the app.
- **Power tools without terminal juggling.** Doctor, Optimizer, Fuzzer,
  Save/RTC Lab, Scenario Recorder, Dev, the Sprite & VRAM profiler,
  evidence-diff, and release packaging remain SDK-owned but are available from
  the native workspace with streaming logs and cancellation.
- **Evidence you can actually review.** SwanSong Studio shows generated controls,
  assets, budgets, native PNG frames, WAV metrics and playback, exact input
  plans, structured evidence, and the resolved SDK, Wonderful, and engine
  identities.
- **Trace a rectangle upstream.** The new bounded source-provenance probe can
  follow selected map, raster, and palette writes through common copies and
  transforms to exact cartridge ranges, then identify other visible consumers.
- **Private details stay private.** Cartridge offsets, emulated addresses,
  per-display chains, and outside-consumer coordinates stay in the selected
  Translation Lab project. MCP receives only hashes, counts, and completeness.
- **Every automation tool in one plugin.** The SwanSong Playtester plugin now
  exposes the two one-shot playtest commands plus persisted capture, both
  rectangle probes, route record/verify, and observed-play start, resume, step,
  finish, and cancel.
- **A real swan in the menu bar.** SwanSong now owns a small swan status item
  with quick Show SwanSong and Quit SwanSong actions.

### SwanSong Studio developer preview

The 0.4 beta intentionally keeps the SDK as a separately testable package.
Choose a local SwanSong SDK checkout and SwanSong Studio invokes its exact `swan`
commands rather than maintaining a second manifest parser or build system.
Compiler and generator diagnostics remain visible in SwanSong.

The public app package does not yet bundle the tagged SDK, Python runtime, or
deterministic play executor. SwanSong Studio therefore remains a developer preview
for people with the SDK checkout and Wonderful toolchain installed. This limit
is shown in the app and is not presented as a self-contained public SDK.

### Before you install

SwanSong requires macOS 14 or later and supports Apple silicon and Intel Macs.
No games, original firmware, SDK toolchain, or commercial game data are
included. Use only game images and project material you own or are authorized
to use.

This beta must be selected explicitly from GitHub Releases. Stable-only updater
clients should not offer it; beta-enabled clients should discover build 6.
Verify the universal app ZIP against `SHA256SUMS.txt` and the matching release
manifest before a manual install.

### Best beta targets

- Create one project from each recipe and run Assets, Test, Profile, and Doctor.
- Exercise Optimizer, Fuzzer, Save/RTC Lab, Scenario Recorder, Dev, Evidence
  Diff, and Release with authorized fixture projects; confirm every structured
  result rejects an unexpected schema.
- Build with the pinned Wonderful toolchain, then run a Play Contract twice and
  inspect the native frame, audio, plan, resource report, and replay status.
- Probe a small translated rectangle with upstream source provenance and verify
  public output contains no cartridge offsets, addresses, coordinates, paths,
  or private artifact names.
- Restart the MCP client and confirm the installed plugin exposes the complete
  fourteen-tool Translation Lab surface alongside the two playtest tools.
- Inspect the SwanSong status item in light and dark menu bars and verify its
  Show and Quit actions.

### Known limits

- Commercial-title compatibility is still being validated. **Reached Video**
  is not an end-to-end verdict or original-hardware accuracy claim.
- Source provenance marks carry-dependent, ambiguous, or overflowing dataflow
  incomplete instead of claiming a false exact dependency.
- SwanSong Studio is not yet a self-contained SDK distribution, as described above.
- The Homebrew Catalog and SwanSong Core download paths remain unavailable
  until their independent signed publication gates pass.
