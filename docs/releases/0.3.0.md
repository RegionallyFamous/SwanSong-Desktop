## SwanSong 0.3.0 beta — Repeatable by Design

SwanSong 0.3 turns Translation Lab work into something you can replay, compare,
and hand off with confidence. It also gives trusted local automation a narrow,
visible door into SwanSong without opening your library or private game data.

### What changed

- **Record a proof route.** Give SwanSong a project and an exact frame/input
  plan and it creates a route-v3 file with a native checkpoint, empty
  persistence, and a fixed proof clock.
- **Verify Original against Patched.** SwanSong replays the same route against
  both project images, captures both native endpoints, runs Capture Intake,
  and writes immutable paired evidence manifests only after both runs finish.
- **Playtest an exact plan.** A bounded deterministic runner can return the
  final native frame, final audio window, complete input trace, and engine
  identity for an authorized local homebrew image.
- **Talk to a running SwanSong.** An opt-in local MCP bridge can read coarse app
  status, navigate sections, and control the already-selected game. Guarded
  MCP tools also expose playtesting and Translation Lab route/evidence work.
- **Use the swan everywhere.** The production icon is now full-bleed and opaque
  for Finder, while direct SwiftPM launches use the compact swan instead of
  AppKit's generic rocket.
- **Start with the fun part.** The README is now a product tour. Detailed
  playing, Translation Lab, automation, Pocket, privacy, build, and release
  documentation lives in the repo-backed Wiki.

### The automation boundary

Local MCP control is off until you enable **Allow local MCP control** in
SwanSong Settings. Enabling it creates a random user-only token for the current
Mac login. Turning it off revokes that token.

The live bridge never returns game titles, paths, ROMs, saves, states, RAM,
screenshots, inputs, or logs. The playtest tool returns image and audio media
only after `confirmShareCapture` is explicitly true. Translation tools require
explicit project-write confirmation, accept only project-contained files, and
return evidence identities rather than private game bytes. A newly configured
Codex client may need to restart before it discovers the project MCP server.

### Still deliberately gated

The first-party Homebrew Catalog remains **Coming Soon**. It has no production
trust key and makes no catalog or game-download request. **Add From Mac** still
works for authorized local homebrew.

The Analogue Pocket SD-card tool remains ready but locked: no authorized stable
SwanSong Core release is currently published. It therefore downloads no Core
package and writes no card. SwanSong Desktop, SwanSong Core, and Sparkle remain
separate release lanes.

### Before you install

SwanSong requires macOS 14 or later and supports Apple silicon and Intel Macs.
No games or original firmware are included. Add only game and homebrew images
you own or are authorized to use.

This beta must be selected explicitly from GitHub Releases. The stable
`/releases/latest` destination remains unchanged. Existing updater-capable
builds see 0.3 only when **Include beta versions** is enabled. Verify the
universal app ZIP against `SHA256SUMS.txt` and the matching release manifest
before a manual install.

### Best beta targets

- Record a short route, replay it repeatedly, and confirm checkpoint and frame
  identities stay stable.
- Verify a route against Original and Patched, then review both immutable
  Capture Intake manifests and their pairing.
- Enable local MCP, test status/navigation/player controls, disable it, and
  confirm the token is revoked immediately.
- Run a bounded homebrew playtest plan and inspect both the final frame and
  final audio window instead of treating a successful command as proof that a
  mechanic works.
- Update from 0.2 with beta versions enabled, then verify library, saves,
  states, preferences, and Translation Lab projects are unchanged.
- Inspect SwanSong in Finder, the Dock, and a direct SwiftPM launch at small and
  large icon sizes.

### Known limits

- Commercial-title compatibility is still being validated. **Reached Video**
  is not an end-to-end verdict or original-hardware accuracy claim.
- MCP is local developer automation, not remote control or a public network
  service. Every mutating tool still requires client approval.
- The Homebrew Catalog and SwanSong Core download paths remain unavailable
  until their independent signed publication gates pass.
- Controllers that macOS does not expose through GameController require a
  compatible driver or mapping layer, or keyboard input.
