## SwanSong 0.4.3 beta — Every Press Counts

SwanSong 0.4.3 makes exact game controls more dependable, brings SwanSong SDK
0.4.0 into Studio, and gives the app's main screens a calmer first read.

Expected app identity: **SwanSong 0.4.3 (9)**.

## Before you install

SwanSong requires macOS 14 or later. The official universal app supports Apple
silicon and Intel Macs. It includes no games or original system firmware; add
only game, homebrew, and project material you own or may use.

This is a beta release. Stable-only clients remain on their selected channel;
beta-enabled clients can discover 0.4.3 after its signed, notarized prerelease
and beta appcast entry are published. For a manual install, verify
`SwanSong-0.4.3-macOS-universal.zip` against `SHA256SUMS.txt` and the matching
release manifest from the same GitHub release.

## Short presses now reach the game on time

SwanSong now sends standard WonderSwan and Pocket Challenge keypad changes into
the live ares input nodes immediately. A short press or release is visible on
its scheduled emulated frame even when the frontend's normal poll happens
later. A clean-room fixture exercises spaced repeated A presses around X1 and
Y3 changes and verifies every transition from the game's own memory trace.

Volume and Power keep ares' normal edge-triggered callback behavior. The fix
does not couple the engine to ares' private `system.controls` internals.

## SwanSong SDK 0.4.0 is built in

Studio embeds the complete tagged SDK 0.4.0 runtime, schemas, recipes, Python
package, documentation, and `swan` entry point. SDK 0.4.0 removes expensive
whole-map and whole-sprite accounting from normal frame presentation, retains
exact profiling on request, improves utility-screen drawing, and strengthens
incremental build dependencies and fresh-boot play contracts.

The app binds the exact SDK commit and normalized content revision, records a
digest for every bundled file, and verifies that identity during build,
packaging, launch, and release checks. Python 3.11+ and Wonderful remain honest
local prerequisites checked by Doctor.

## Clearer screens

Library, Homebrew, Translation Lab, Analogue Pocket, Studio, Settings, Updates,
and Support now lead with plain-language headlines and obvious next actions.
Game cards distinguish a picture appearing, the player's own verdict, play
readiness, and file health. Technical proof remains available without taking
over the first read.

## Boundaries and known limits

- SwanSong remains the only automated emulator and gameplay-validation backend.
- Studio invokes the SDK rather than reimplementing manifests, assets, builds,
  budgets, evidence, or release policy.
- Python, Wonderful, and project-declared Wonderful packages are not bundled.
- Commercial-title compatibility remains under active validation. **Reached
  Video** is not an end-to-end gameplay verdict or an original-hardware claim.
- No games, original firmware, private evidence, or translation material are
  included in the app.

When reporting a beta issue, include the app build, macOS version, Mac
architecture, controller when relevant, and sanitized result. Do not attach
ROMs, saves, private screenshots/audio, project paths, or Translation Lab
evidence.
