## SwanSong 0.6.0 — Make the Whole Workshop Move

SwanSong 0.6.0 turns the tools around the player into places you can really
work. Studio gets the complete SwanSong SDK 0.5 toolset, Story Forge grows from
a release dashboard into a living writing room, and Translation Lab gains a
stronger way to prove that private source evidence belongs to the exact frame
you captured.

Expected app identity: **SwanSong 0.6.0 (11)**.

## Before you install

SwanSong requires macOS 14 or later. The official universal app supports Apple
silicon and Intel Macs. It includes no games or original system firmware; use
only game, manuscript, project, and cartridge material you own or may handle.

This is a stable release. Existing SwanSong users can discover 0.6.0 through
the normal update channel after its signed appcast entry is published. For a
manual install, verify `SwanSong-0.6.0-macOS-universal.zip` against
`SHA256SUMS.txt` and the matching release manifest from the same GitHub
release.

## SwanSong Studio is ready for bigger projects

The signed app now carries the complete, content-verified SwanSong SDK 0.5.0
toolset. Create Utility Apps, run trace-aware builds, compile scenarios, inspect
semantic outcomes, review asset and audio work, follow budget history, migrate
projects, and take an evidence-backed release all the way through one native
workspace.

The local automation surface grows with the same care. Safe, path-free SDK
actions can help with all-contract play, optimizer preview, fuzzing, Save/RTC
Lab, one-shot Dev, migration preview, and hardware capacity. File selection,
destructive migration, asset changes, and Release stay visible Mac actions.

## Story Forge feels like a real writing room

Story Forge now has nine focused workspaces. A proposal-only Story Room prepares
fresh packets for eight specialists without allowing any of them to rewrite the
book or approve their own work. The new story map makes causality, relationships,
setups, payoffs, rhythm, art, and music easier to see together.

You can edit chapters in SwanSong with explicit Save and Discard actions,
refresh live context for a stable scene ID, preserve immutable revision
snapshots, and append the reason you accepted, partly used, or rejected a
change. Reader Lab keeps reactions unprimed. Research connects claims and
sources to scenes. Genre review asks sharper questions without turning taste
into a formula.

Art Room keeps ImageGen prompts and intake history beside the work. Music Room
creates editable four-channel sketches and renders two mono auditions with
useful measurements. The adaptation workspace builds a source-mapped `.wscvn`
scaffold, shows drift when the novel or game plan changes, and hands the real
production pass to SwanSong Studio.

## Translation evidence is bound before it is traced

A new guarded capture-plan envelope can authenticate the project, frame plan,
Original and Patched ROMs, private capture directory, and exact allowed outputs
before it performs the bounded record, replay, capture, and intake sequence.
The result stays inside the Translation Lab project.

Capture-bound source probes now validate the expected native frame number and
framed pixel fingerprint before asking the engine for owner or source data.
They check the frame again afterward and retain an ordered private receipt. A
mismatch stops before provenance begins and leaves no partial report, details,
run directory, or closure behind.

Capture Intake also gets a tighter boundary: one fresh private directory,
exactly two expected outputs, and a verified receipt. Toolkit launches are
bound to the selected executable, entry point, working directory, arguments,
and a small deterministic environment while public automation receives only a
redacted digest summary.

## A calmer, more repeatable build

Trusted local MCP and Swift package launchers no longer need the login keychain,
which removes an unexpected password prompt from ordinary development and
automation. The prepared ares source is sealed during compilation, and a new
reproducibility gate builds it twice in separate private roots before comparing
the universal engine, exported ABI, source identity, monochrome output, and
route-runner binding.

The private provenance fixtures now cover monochrome palette output and mapper
windows more thoroughly. Long Story Forge route matrices show live progress and
wall time, and a timeout ends the complete child process group instead of
leaving a hidden player behind.

## Boundaries and known limits

- SwanSong Studio still needs Python 3.11+ and the pinned Wonderful packages
  for development work; Doctor explains what is missing.
- Story Forge requires a compatible local schema-v3 framework checkout.
- Rectangle source probing keeps exact cartridge ranges, addresses, writers,
  lineages, and outside consumers private. Automation receives source-free
  hashes, counts, geometry, and completeness only.
- The separate Analogue Pocket Core remains unavailable until its own project
  publishes an authorized stable release.
- Commercial-title compatibility remains under active validation. **Reached
  Video** is not an end-to-end gameplay or original-hardware verdict.
- No games, original firmware, private evidence, manuscripts, or cartridge
  contents are included in the app.

When reporting an issue, include SwanSong 0.6.0 build 11, macOS version, Mac
architecture, and the visible action and message. Never attach ROMs, saves,
private screenshots or audio, manuscripts, project paths, bearer tokens, or
Translation Lab evidence.
