## SwanSong 0.3.1 beta — See What Changed

SwanSong 0.3.1 makes Translation Lab feel less like a collection of tools and
more like a workspace that remembers the whole investigation. Capture a change,
inspect where the pixels came from, play forward at your own pace, and come back
later without losing the proof SwanSong gathered along the way.

### What changed

- **Keep every comparison.** Give SwanSong one exact plan and it privately saves
  the Original and Patched native frames, the plan that reached them, their
  deterministic bindings, and the exact pixel difference inside the project.
- **Ask who drew a rectangle.** The new display-owner probe can trace a selected
  native area back through active layers, map cells, tiles or raster data,
  palettes, and the final writer. Detailed evidence stays private; automation
  receives only safe hashes and counts.
- **Play one visible step at a time.** Observed-play sessions let trusted local
  automation advance a project in bounded steps while SwanSong continuously
  records the complete from-boot plan. Final evidence is still proven by a clean
  deterministic replay.
- **Pick up after an interruption.** If the automation host stops, SwanSong marks
  the session interrupted and reconstructs it only by validating and replaying
  the saved plan from boot.
- **Browse the evidence.** Translation Lab now shows saved pairs, probes, and
  observed sessions with integrity and size information, safe Finder reveal,
  source-free JSON export, guarded deletion, and low-disk protection.
- **Compare from the playtest tool.** One guarded command can replay the same
  exact plan against separate Original and Patched images and return labeled
  native captures plus whole-frame pixel and audio differences.

### Private by design

The new evidence lives inside the Translation Lab project you chose. Rectangle
source addresses, map cells, palette values, writer identities, ROM data, save
data, persistence, and RAM are not returned through MCP. Media is shared only
after the caller explicitly confirms it. Durable writes are bounded, checked
for available space, and published atomically so a partial run does not look
like completed evidence.

### Before you install

SwanSong requires macOS 14 or later and supports Apple silicon and Intel Macs.
No games or original firmware are included. Add only game and homebrew images
you own or are authorized to use.

This beta must be selected explicitly from GitHub Releases. The stable
`/releases/latest` destination remains unchanged. Existing updater-capable
builds see 0.3.1 only when **Include beta versions** is enabled. Verify the
universal app ZIP against `SHA256SUMS.txt` and the matching release manifest
before a manual install.

### Best beta targets

- Persist a short Original/Patched capture and confirm it survives an app
  restart with both frames, the plan, and the diff intact.
- Probe a small changed rectangle and confirm detailed ownership stays in the
  project while the automation response contains only hashes and counts.
- Advance an observed session beyond the old replay limit, interrupt its MCP
  host, resume it, and finish with clean-boot paired proof.
- Browse, export a source-free summary of, and delete a disposable evidence
  artifact; confirm an active session cannot be deleted.
- Exercise low-disk preflight and verify it blocks unsafe durable writes without
  damaging existing evidence.

### Known limits

- Commercial-title compatibility is still being validated. **Reached Video**
  is not an end-to-end verdict or original-hardware accuracy claim.
- Display-owner results describe the exact rendered frame and rectangle. A save
  state restored from an older point cannot honestly claim the original CPU
  writer provenance, so SwanSong marks that detail unavailable.
- MCP is local developer automation, not remote control or a public network
  service. Every mutating tool still requires client approval.
- The Homebrew Catalog and SwanSong Core download paths remain unavailable
  until their independent signed publication gates pass.
