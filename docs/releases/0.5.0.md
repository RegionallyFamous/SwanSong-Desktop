## SwanSong 0.5.0 — From Story to Cartridge

SwanSong 0.5.0 opens two new doors. Story Forge turns the Mac app into a
thoughtful writing and publishing desk. Cartridge Lab connects SwanSong to a
real WonderSwan Color or SwanCrystal for careful, verifiable preservation work.

Expected app identity: **SwanSong 0.5.0 (10)**.

## Before you install

SwanSong requires macOS 14 or later. The official universal app supports Apple
silicon and Intel Macs. It includes no games or original system firmware; use
only game, manuscript, project, and cartridge material you own or may handle.

This is a stable release. Existing SwanSong users can discover 0.5.0 through
the normal update channel after its signed appcast entry is published. For a
manual install, verify
`SwanSong-0.5.0-macOS-universal.zip` against `SHA256SUMS.txt` and the matching
release manifest from the same GitHub release.

## A writing desk inside SwanSong

Story Forge carries a light novel from first idea through outline, draft,
revision, art, continuity, rights, and release. It can refresh editorial and
reader reports, manage a multi-book catalog, prepare ImageGen illustration
briefs, check an optional soundtrack plan, lock a manuscript, and build EPUB
and PDF editions.

The app never lets a score impersonate an editor. Human approval stays human,
private manuscripts stay in folders you choose, and a revision-ready novel can
move directly into SwanSong Studio for a WonderSwan adaptation.

Story Forge uses the separate schema-v3 Story Forge framework checkout you
select. SwanSong exposes its reviewed commands through fixed native actions;
it does not run arbitrary project commands or copy the framework into the app.

## The real handheld joins the preservation loop

Cartridge Lab uses a WonderSwan Color or SwanCrystal as its own cartridge
reader. With Yokoi Boot and a 3.3 V ExtFriend-compatible USB serial adapter,
the Mac app can inspect an inserted cartridge, create checksum-verified ROM and
SRAM/EEPROM backups, and restore an exact-size save only after confirmation on
both the Mac and the console.

Backups are written to a temporary sibling, checked as they arrive, and
promoted only when complete. Save restoration stays disarmed until physical
A+B confirmation and ends with a complete readback. Retail mask ROM is always
read-only.

Cartridge Lab can also add the verified Yokoi Boot installer to a compatible
flash-cartridge SD folder without replacing a different file. The first setup
still needs a cartridge that can already launch `.wsc` software and provides
at least 8 KiB of SRAM, a WonderWitch route, or direct EEPROM programming. Never
connect the EXT port to PC RS-232 voltage.

## A better first read

The README, installation guide, FAQ, Support, privacy material, and Wiki now
lead with what SwanSong lets you do. Playing, translating, making, writing, and
preserving each have a clear path, while the exact technical and release
contracts remain available when you want the proof.

## Homebrew works in the signed app

The Homebrew Catalog anti-rollback record now uses the protected macOS Keychain
available to directly distributed Developer ID apps. The notarized app can open
the signed first-party catalog without asking for an entitlement it does not
carry. Catalog loading, refreshing, and downloads remain explicit actions.

## Boundaries and known limits

- Story Forge requires a compatible local schema-v3 framework checkout.
- Cartridge Lab requires authorized physical hardware and is limited to
  WonderSwan Color and SwanCrystal.
- A stock console cannot receive its first Yokoi Boot loader through EXT alone.
- Python and Wonderful remain local SwanSong Studio prerequisites.
- The separate Analogue Pocket Core is still unavailable until its own project
  publishes an authorized stable release.
- Commercial-title compatibility remains under active validation. **Reached
  Video** is not an end-to-end gameplay or original-hardware verdict.
- No games, original firmware, private evidence, manuscripts, or cartridge
  contents are included in the app.

When reporting a beta issue, include the app build, macOS version, Mac
architecture, controller or hardware setup when relevant, and a sanitized
result. Never attach ROMs, saves, private screenshots/audio, manuscripts,
project paths, or Translation Lab evidence.
